# Vite 调试环境搭建完整指南

## 第一部分：顶层鸟瞰

### 背景问题

在前面的课程中，我们使用 Rollup 打包 + pnpm link 的方式进行测试，这种方式存在以下问题：

**核心挑战：**
- 每次修改代码都需要重新打包
- 无法实时看到源码运行效果
- 调试效率低下

**解决方案：**
使用 Vite 搭建实时调试环境，通过模块别名（alias）直接引用源码，实现热更新和实时调试。

### 宏观图

**Vite 调试环境架构：**

```
Vite 项目（demos/test-fc）
    ↓
vite.config.js (配置 alias)
    ↓
alias 映射
    ├─ react → packages/react/index.ts
    ├─ react-dom/client → packages/react-dom/client.ts
    └─ hostConfig → packages/react-dom/src/hostConfig.ts
    ↓
直接使用源码（无需打包）
    ↓
HMR 热更新（实时预览）
```

**核心优势：**
1. **实时调试** - 修改源码立即生效
2. **无需打包** - 直接运行 TypeScript 源码
3. **提升效率** - 大幅缩短开发反馈周期

### 关键技术栈

| 技术 | 说明 |
|------|------|
| **Vite** | 现代化前端构建工具，支持快速热更新 |
| **@vitejs/plugin-react** | Vite 的 React 插件，支持 JSX |
| **Module Alias** | 模块别名配置，映射源码路径 |
| **TypeScript** | 直接运行 TS 源码 |

## 第二部分：实战步骤

### 步骤 1：创建 Vite 项目

**路径：** `big-react` 项目根目录

在项目根目录下执行以下命令创建 Vite 项目：

```bash
pnpm create vite
```

创建时的配置选项：
- Project name: `demos`
- Select a framework: `React`
- Select a variant: `TypeScript`

### 步骤 2：创建测试项目目录

**路径：** `demos/test-fc`

在 demos 目录下创建 `test-fc` 子目录，用于存放 FunctionComponent 的测试代码。

### 步骤 5：配置 vite.config.js

**路径：** `scripts/vite/vite.config.js`

**修改内容：**
1. 导入必要的依赖
2. 配置 react 插件
3. 配置 resolve.alias 映射源码路径

```javascript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import replace from '@rollup/plugin-replace';
import { resolvePkgPath } from '../rollup/utils';

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    react(),
    replace({
      __DEV__: true,
      preventAssignment: true
    })
  ],
  resolve: {
    alias: [
      {
        find: 'react',
        replacement: resolvePkgPath('react')
      },
      {
        find: 'react-dom',
        replacement: resolvePkgPath('react-dom')
      },
      {
        find: 'hostConfig',
        replacement: path.resolve(
          resolvePkgPath('react-dom'),
          './src/hostConfig.ts'
        )
      }
    ]
  }
});
```

### 步骤 11：启动 Vite 开发服务器

**路径：** `big-react` 项目根目录

```bash
pnpm demo
```

执行后，Vite 会启动开发服务器，通常在 `http://localhost:5173` 端口。

打开浏览器访问该地址，可以看到：
- 页面显示 "big-react" 文本
- 控制台输出 `1234`（如果添加了步骤10的调试代码）
- 控制台显示 commit 阶段和 Placement 操作的日志

**验证热更新：**
- 修改 `main.tsx` 中的文本内容
- 保存文件后，浏览器会自动刷新并显示最新内容
- 无需手动刷新或重新打包
