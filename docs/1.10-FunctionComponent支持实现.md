# FunctionComponent 支持实现完整指南

## 第一部分：顶层鸟瞰

### 背景问题

在前面的实现中，我们已经完成了 HostComponent（原生DOM组件）和 HostText（文本节点）的渲染。现在需要支持 FunctionComponent，使得 React 能够处理函数组件。

**核心挑战：**
- 如何在 beginWork 阶段处理 FunctionComponent
- 如何执行函数组件并获取其返回的 children
- 如何在 completeWork 阶段处理 FunctionComponent

### 宏观图

**FunctionComponent 的处理流程：**

```
beginWork
    ↓
case FunctionComponent
    ↓
updateFunctionComponent
    ↓
renderWithHooks (执行函数组件)
    ↓
reconcileChildren (协调子节点)
    ↓
completeWork
    ↓
case FunctionComponent
    ↓
bubbleProperties (向上冒泡副作用)
```

### 关键技术栈

| 技术 | 说明 |
|------|------|
| **TypeScript** | 类型定义（FiberNode） |
| **Fiber 架构** | FunctionComponent 的 beginWork 和 completeWork |
| **Hooks 机制** | renderWithHooks 函数执行组件 |
| **Reconciler** | reconcileChildren 协调子节点 |

## 第三部分：主要知识点

### 1. FunctionComponent 需要考虑的问题

在实现 FunctionComponent 支持时，需要重点考虑两个核心问题：

#### 如何支持 FC？

FunctionComponent 的工作同样植根于 Fiber 架构的两个核心阶段：

- **beginWork 阶段**：需要执行函数组件，获取其返回的 JSX 结构（children），然后进行 reconcile
- **completeWork 阶段**：FunctionComponent 本身不产生 DOM 节点，只需要向上冒泡子节点的副作用

#### 如何组织 Hooks？

Hooks 机制将在下一节课详细讲解。目前的 `renderWithHooks` 函数只是一个基础实现，用于执行函数组件：

```typescript
export function renderWithHooks(wip: FiberNode) {
  const Component = wip.type;  // 获取函数组件
  const props = wip.pendingProps;  // 获取 props
  const children = Component(props);  // 执行函数组件
  return children;  // 返回 children
}
```

### 2. 第二种调试方式

在开发过程中，除了使用传统的打包方式，还可以采用 **Vite 的实时调试**。

**优点：** "实时看到源码运行效果"

使用 Vite 时，源码的修改会立即反映到浏览器中，无需重新打包，大大提升了开发效率。
