# React DOM 包实现与打包配置完整指南

## 第一部分：顶层鸟瞰

### 1. 背景问题

在前面的章节中，我们已经完成了 `react-reconciler` 的核心逻辑实现，但它是平台无关的。现在需要实现 `react-dom` 包，将 reconciler 与浏览器 DOM 环境连接起来，并完成打包配置，让代码能够在浏览器中运行。

本章要解决的核心问题：
- **如何实现浏览器平台特定的 DOM 操作？**（`hostConfig`）
- **如何兼容原版 React 的导出接口？**（处理 `shared` 和 `react-reconciler` 的依赖）
- **如何配置 Rollup 打包？**（别名、路径解析、TypeScript 配置）
- **如何实现开发环境的快速构建？**（`dev.config.js`）

### 2. 宏观流程图

```
react-dom 包结构
    ↓
├── package.json              ← 包配置（dependencies）
├── src/
│   ├── hostConfig.ts         ← 浏览器 DOM 操作（createInstance, appendChild...）
│   ├── root.ts               ← createRoot API（调用 reconciler）
│   └── index.ts              ← 默认导出（兼容原版 React）
└── scripts/rollup/
    ├── react-dom.config.js   ← react-dom 打包配置（alias, generatePackageJson）
    ├── dev.config.js         ← 开发环境配置（合并 react 和 react-dom）
    └── utils.js              ← 工具函数（getBaseRollupPlugins）

打包流程
    ↓
1. hostConfig 处理指向        ← alias 将 hostConfig 指向具体实现
2. 原版 React 导出兼容        ← 处理 shared/react-reconciler 依赖
3. TypeScript 编译            ← 根 tsconfig.json：noUnusedParameters: false
4. 生成 package.json          ← generatePackageJson（peerDependencies）
5. 输出到 dist 目录           ← index.js + client.js
```

### 3. 关键技术栈

| 技术点 | 说明 |
|--------|------|
| **hostConfig 指向处理** | 通过 Rollup alias 将抽象的 hostConfig 指向具体的 react-dom/src/hostConfig.ts |
| **原版 React 导出兼容** | 兼容 `shared` 和 `react-reconciler` 包，使其在 `workspace:*` 依赖下正确工作 |
| **generatePackageJson** | 动态生成 dist/package.json，添加 `peerDependencies: { react: version }` |
| **TypeScript noUnusedParameters** | 在根 tsconfig.json 中设置为 `false`，允许 hostConfig 中的未使用参数（如 `props: any`）|
| **开发环境配置** | 使用 `dev.config.js` 合并 react 和 react-dom 配置，加速开发构建 |

  child: Instance

### 步骤 6：新建 react-dom 打包配置

**路径：** `scripts/rollup/react-dom.config.js`

**修改内容**：
1. 导入工具函数 `getPackageJSON`, `resolvePkgPath`, `getBaseRollupPlugins`
2. 导入 `generatePackageJson` 插件
3. 配置 react-dom 包路径和产物路径
4. 配置两个输出：`index.js`（UMD 格式）和 `client.js`（UMD 格式）
5. 在 plugins 中添加 `alias` 配置（映射 `hostConfig`）
6. 在 plugins 中添加 `generatePackageJson` 配置（生成 peerDependencies）

```javascript
import { getPackageJSON, resolvePkgPath, getBaseRollupPlugins } from './utils';
import generatePackageJson from 'rollup-plugin-generate-package-json';
import alias from '@rollup/plugin-alias';

const { name, module } = getPackageJSON('react-dom');
// react-dom包的路径
const pkgPath = resolvePkgPath(name);
// react-dom产物路径
const pkgDistPath = resolvePkgPath(name, true);

export default [
  // react-dom
  {
    input: `${pkgPath}/${module}`,
    output: [
      {
        file: `${pkgDistPath}/index.js`,
        name: 'index.js',
        format: 'umd'
      },
      {
        file: `${pkgDistPath}/client.js`,
        name: 'client.js',
        format: 'umd'
      }
    ],
    plugins: [
      ...getBaseRollupPlugins(),
      // webpack resolve alias
      alias({
        entries: {
          hostConfig: `${pkgPath}/src/hostConfig.ts`
        }
      }),
      generatePackageJson({
        inputFolder: pkgPath,
        outputFolder: pkgDistPath,
        baseContents: ({ name, description, version }) => ({
          name,
          description,
          version,
          peerDependencies: {
            react: version
          },
          main: 'index.js'
        })
      })
    ]
  }
];
```

## 第三部分：主要知识点

### 1. 打包 ReactDOM 的注意事项

根据截图 1-1，打包 ReactDOM 需要注意以下两点：

#### 1.1 兼容原版 React 的导出

- **问题**：原版 React 使用了 `shared` 和 `react-reconciler` 包，而我们的实现也有这些包，但它们是通过 `workspace:*` 依赖的。
- **解决方案**：
  - 在 `package.json` 中正确配置 `dependencies` 和 `peerDependencies`
  - 使用 `generatePackageJson` 插件生成正确的 `dist/package.json`

#### 1.2 处理 hostConfig 的指向

- **问题**：`react-reconciler` 中的 `hostConfig` 是抽象的，需要指向具体的平台实现（如 `react-dom/src/hostConfig.ts`）
- **解决方案**：
  - 在 Rollup 配置中使用 `alias` 插件映射路径
  - **关键配置**：
    ```javascript
    alias({
      entries: {
        hostConfig: `${pkgPath}/src/hostConfig.ts`
      }
    })
    ```

### 2. TypeScript 配置

- **noUnusedParameters: false**：允许参数暂时未使用；当前 `createInstance` 用注释保留了 `props` 占位，后续可逐步启用透传。
- **noUnusedLocals: false**：允许局部变量未使用

### 3. Rollup 打包配置

- **输出格式**：使用 `umd` 格式，同时生成 `index.js` 和 `client.js`
- **generatePackageJson**：自动生成 `dist/package.json`，包含 `peerDependencies`
- **alias 插件**：解决 `hostConfig` 的路径映射问题

### 4. 开发环境优化

- **dev.config.js**：合并 react 和 react-dom 的配置，一次构建两个包
- **build:dev 脚本**：快速构建开发版本，用于测试
