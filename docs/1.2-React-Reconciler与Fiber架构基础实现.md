### 步骤 3：实现 FiberNode 类

FiberNode 是 React Fiber 架构的核心数据结构，用于表示虚拟DOM树中的一个节点。

#### 3.1 创建 fiber.ts

**路径：** `packages/react-reconciler/src/fiber.ts`

**完整实现：**

```typescript
import { Props, Key, Ref } from 'shared/ReactTypes';
import { WorkTag } from './workTags';
import { Flags, NoFlags } from './fiberFlags';

/**
 * FiberNode - React Fiber 架构的核心数据结构
 * 
 * ⭐ 用于表示虚拟DOM树中的一个节点
 * ⭐ 介于 React Element 与真实 UI 节点之间
 * ⭐ 不仅作为数据存储单元，也能作为工作单元
 */
export class FiberNode {
  // ==================== 1. 实例属性（标识节点）====================
  
  /**
   * tag - 节点类型标记
   * ⭐ 类型：WorkTag
   * ⭐ 说明：标识节点类型（FunctionComponent、HostComponent等）
   */
  tag: WorkTag;
  
  /**
   * type - 节点的具体类型
   * ⭐ 类型：any
   * ⭐ 说明：函数组件是函数、DOM节点是标签名（如 'div'）
   */
  type: any;
  
  /**
   * pendingProps - 即将应用的新 props
   * ⭐ 类型：Props
   * ⭐ 说明：更新时的新 props
   */
  pendingProps: Props;
  
  /**
   * key - React key 属性
   * ⭐ 类型：Key
   * ⭐ 说明：用于列表diff优化
   */
  key: Key;
  
  /**
   * stateNode - 对应的真实节点
   * ⭐ 类型：any
   * ⭐ 说明：HostComponent 对应真实 DOM，FunctionComponent 对应组件实例
   */
  stateNode: any;
  
  /**
   * ref - React ref 引用
   * ⭐ 类型：Ref
   * ⭐ 说明：用于获取 DOM 节点或组件实例
   */
  ref: Ref;
  
  // ==================== 2. 树状结构属性（构成 Fiber 树）====================
  
  /**
   * return - 父节点指针
   * ⭐ 类型：FiberNode | null
   * ⭐ 说明：指向父 Fiber 节点
   */
  return: FiberNode | null;
  
  /**
   * sibling - 右兄弟节点指针
   * ⭐ 类型：FiberNode | null
   * ⭐ 说明：指向下一个兄弟 Fiber 节点
   */
  sibling: FiberNode | null;
  
  /**
   * child - 第一个子节点指针
   * ⭐ 类型：FiberNode | null
   * ⭐ 说明：指向第一个子 Fiber 节点
   */
  child: FiberNode | null;
  
  /**
   * index - 在父节点中的索引
   * ⭐ 类型：number
   * ⭐ 说明：标识在父节点children中的位置
   */
  index: number;
  
  // ==================== 3. 工作单元属性 ====================
  
  /**
   * memoizedProps - 上一次渲染时的 props
   * ⭐ 类型：Props | null
   * ⭐ 说明：用于对比新旧 props
   */
  memoizedProps: Props | null;
  
  /**
   * alternate - 双缓冲技术对应节点
   * ⭐ 类型：FiberNode | null
   * ⭐ 说明：current ↔ workInProgress 互相指向
   */
  alternate: FiberNode | null;
  
  /**
   * flags - 副作用标记
   * ⭐ 类型：Flags
   * ⭐ 说明：标记节点的副作用（Placement、Update、Deletion等）
   */
  flags: Flags;
  
  constructor(tag: WorkTag, pendingProps: Props, key: Key) {
    // ========== 实例属性初始化 ==========
    this.tag = tag;
    this.key = key;
    // HostComponent <div> → div DOM
    this.stateNode = null;
    // FunctionComponent → () => {}
    this.type = null;
    
    // ========== 构成树状结构 ==========
    this.return = null;     // 父节点
    this.sibling = null;    // 右兄弟节点
    this.child = null;      // 第一个子节点
    this.index = 0;         // 在父节点中的索引
    
    this.ref = null;
    
    // ========== 作为工作单元 ==========
    this.pendingProps = pendingProps;
    this.memoizedProps = null;
    this.alternate = null;   // 双缓冲技术：current ↔ workInProgress
    this.flags = NoFlags;    // 初始无副作用
  }
}
```

**Fiber 树的结构示例：**

```
        div (父节点)
         |
    ┌────┴────┐
    |         |
   h3      sibling → p
    |                |
  child           child
    |                |
  "你好"        "Big-React"
```

通过 `return`、`child`、`sibling` 三个指针，构成了完整的 Fiber 树结构。

import { WorkTag } from './workTags';
import { Flags, NoFlags } from './fiberFlags';

export class FiberNode {
  // ... 其他属性
  
  alternate: FiberNode | null;
  flags: Flags;

    // ... 其他初始化代码

    this.flags = NoFlags;

**双缓冲技术（alternate）：**
- React 维护两棵 Fiber 树：
  - **current**：当前屏幕上显示的 Fiber 树
  - **workInProgress**：正在内存中构建的 Fiber 树
- 每个节点通过 `alternate` 指针指向另一棵树中对应的节点
- 渲染完成后，直接切换两棵树的指针，实现快速更新

### 步骤 5：实现工作循环（Work Loop）

#### 5.1 创建 workLoop.ts 的基础结构

**路径：** `packages/react-reconciler/src/workLoop.ts`

```typescript
import { FiberNode } from './fiber';
import { beginWork } from './beginWork';
import { completeWork } from './completeWork';

/**
 * workInProgress - 当前正在处理的 Fiber 节点
 * ⭐ 核心变量：追踪当前工作的 Fiber 节点
 */
let workInProgress: FiberNode | null = null;

/**
 * prepareFreshStack - 初始化工作栈
 * 
 * @param fiber - 根节点
 * 
 * ⭐ 作用：将根节点设为当前工作节点
 */
function prepareFreshStack(fiber: FiberNode) {
  workInProgress = fiber;
}

/**
 * renderRoot - 渲染入口函数
 * 
 * @param root - 根 Fiber 节点
 * 
 * ⭐ 作用：开始渲染流程
 */
function renderRoot(root: FiberNode) {
  // 初始化
  prepareFreshStack(root);
}
```

#### 5.2 实现工作循环主体

```typescript
/**
 * renderRoot - 渲染入口函数
 * 
 * ⭐ 工作循环流程：
 * 1. 初始化 workInProgress 为根节点
 * 2. 进入 do...while(true) 循环，执行 workLoop
 * 3. workLoop 循环处理每个 Fiber 节点
 * 4. 如果发生错误，捕获异常并重置 workInProgress
 * 5. 正常完成时，跳出循环
 */
function renderRoot(root: FiberNode) {
  // 初始化
  prepareFreshStack(root);
  
  do {
    try {
      workLoop();
      break;
    } catch (e) {
      console.warn('workLoop发生错误', e);
      workInProgress = null;
    }
  } while (true);
}

/**
 * workLoop - 工作循环
 * 
 * ⭐ 核心循环：不断处理 workInProgress 节点，直到为 null
 */
function workLoop() {
  while (workInProgress !== null) {
    performUnitOfWork(workInProgress);
  }
}
```

#### 5.3 实现 performUnitOfWork（递阶段）

```typescript
/**
 * performUnitOfWork - 处理单个工作单元（递阶段）
 * 
 * @param fiber - 当前 Fiber 节点
 * 
 * ⭐ 作用：
 * - 对当前 Fiber 节点执行 beginWork（递阶段）
 * - 将 pendingProps 赋值给 memoizedProps（确认 props）
 * - 如果有子节点（next），继续向下遍历
 * - 如果没有子节点，进入 completeUnitOfWork（归阶段）
 */
function performUnitOfWork(fiber: FiberNode) {
  const next = beginWork(fiber);           // 执行递阶段，返回子节点
  fiber.memoizedProps = fiber.pendingProps; // 确认 props
  
  if (next === null) {
    // 没有子节点，开始归阶段
    completeUnitOfWork(fiber);
  } else {
    // 有子节点，继续递阶段
    workInProgress = next;
  }
}
```

#### 5.4 实现 completeUnitOfWork（归阶段）

```typescript
/**
 * completeUnitOfWork - 完成工作单元（归阶段）
 * 
 * @param fiber - 当前 Fiber 节点
 * 
 * ⭐ 流程：
 * 1. 对当前节点执行 completeWork（归阶段处理）
 * 2. 检查是否有兄弟节点（sibling）：
 *    - 如果有，将 workInProgress 指向兄弟节点，返回
 *    - 如果没有，向上返回父节点（return）
 * 3. 重复步骤 1-2，直到回到根节点
 */
function completeUnitOfWork(fiber: FiberNode) {
  let node: FiberNode | null = fiber;
  
  do {
    completeWork(node);              // 执行归阶段
    const sibling = node.sibling;
    
    if (sibling !== null) {
      // 有兄弟节点，处理兄弟节点
      workInProgress = sibling;
      return;
    }
    // 没有兄弟节点，返回父节点
    node = node.return;
    workInProgress = node;
  } while (node !== null);
}
```

**完整的遍历流程（DFS深度优先遍历）：**

```
递（beginWork）：
  <Card>
    → <h3>你好</h3>
      → "你好" (文本节点)
      
归（completeWork）：
      ← "你好"
    ← <h3>
    → <p>Big-React</p>  (处理兄弟节点)
      → "Big-React"
      ← "Big-React"
    ← <p>
  ← <Card>
```

## 第三部分：主要知识点

### 知识点 1：React 架构概述

#### 前端框架结构与工作原理（状态驱动）

```
┌─────────────────┐
│  描述UI的方式    │
│  ┌──────────┐   │   编译优化   ┌────────────────┐
│  │   JSX    │   ├──────────────→│   运行时核心   │
│  └──────────┘   │               │  ┌──────────┐  │
│  ┌──────────┐   │               │  │reconciler│  │
│  │ 组件语法  │   │               │  └──────────┘  │
│  └──────────┘   │               │  ┌──────────┐  │
│                 │               │  │ renderer │  │
└─────────────────┘               │  └──────────┘  │
                                  └────────────────┘
```

**架构说明：**

| 模块 | 说明 |
|------|------|
| **描述UI的方式** | JSX 语法 + 组件化 |
| **编译优化** | Babel 将 JSX 编译成 React.createElement 或 jsx() 调用 |
| **运行时核心模块** | 包括 reconciler（协调器）和 renderer（渲染器） |
| **reconciler** | 负责 Fiber 树的构建、Diff 算法、调度 |
| **renderer** | 负责将 Fiber 树渲染到不同平台（DOM、Native、Canvas等） |

**工作流程：**
1. 开发者使用 JSX 描述 UI
2. Babel 编译 JSX 为函数调用
3. React 通过 reconciler 构建/更新 Fiber 树
4. renderer 根据 Fiber 树渲染到目标平台

### 知识点 2：为什么需要 FiberNode？

#### React Element 的局限性

**当前已知的数据结构：React Element（JSX转换 playground）**

React Element 作为核心模块操作的数据结构存在以下问题：

**React Element 的问题：**
- 无法表达节点之间的关系
- 字段有限，不好拓展（比如：无法表达状态）

#### FiberNode 的优势

所以，需要一种新的数据结构，它的特点：
- **介于 React Element 与真实 UI 节点之间**
- **能够表达节点之间的关系**
- **方便拓展**（不仅作为数据存储单元，也能作为工作单元）

**这就是 FiberNode（虚拟DOM在React中的实现）**

---

### 知识点 3：React 的节点类型

#### 当前我们了解的节点类型：

1. **JSX**
   - 开发者编写的 JSX 语法
   - 例如：`<div>hello</div>`

2. **React Element**
   - JSX 编译后的对象
   - 例如：`{ $$typeof: Symbol(react.element), type: 'div', props: {...} }`

3. **FiberNode**
   - React 内部的虚拟 DOM 节点
   - 包含树状结构、副作用标记、工作单元信息

4. **DOM Element**
   - 真实的 DOM 节点
   - 例如：`document.createElement('div')`

### 知识点 4：Reconciler 的工作方式

#### 双缓冲技术

对于同一个节点，比较其 **React Element** 与 **fiberNode**，生成子 **fiberNode**。并根据比较的结果生成不同标记（插入、删除、移动......），对应不同宿主环境API的执行。

**双缓冲原理：**

当所有 React Element 比较完后，会生成一棵 **fiberNode树**，一共存在两棵 fiberNode 树：

- **current**：与视图中真实UI对应的 fiberNode树
- **workInProgress**：触发更新后，正在 reconciler 中计算的 fiberNode树

**示例：挂载 `<div></div>`**

```javascript
// React Element <div>
jsx("div")
// 对应fiberNode
null
// 生成子fiberNode
// 对应标记
Placement
```

**示例：将 `<div></div>` 更新为 `<p></p>`**

```javascript
// React Element <p></p>
jsx("p")
// 对应fiberNode
FiberNode {type: 'div'}
// 生成子fiberNode
// 对应标记
Deletion Placement
```

**说明：**
- 首次渲染时，current 为 null，生成新的 fiberNode 并标记 Placement
- 更新时，比较新的 React Element 与 current fiberNode，生成 workInProgress fiberNode
- 根据比较结果，标记不同的副作用（Deletion、Placement、Update等）

### 知识点 5：JSX 消费的顺序

#### DFS 深度优先遍历与 BFS 广度优先遍历详解

以 DFS（深度优先遍历）的顺序遍历 React Element，这意味着：

- **如果有子节点，遍历子节点**
- **如果没有子节点，遍历兄弟节点**

**示例：**

```jsx
<Card>
  <h3>你好</h3>
  <p>Big-React</p>
</Card>
```

**这是个递归的过程，存在递、归两个阶段：**

- **递**：对应 **beginWork**
- **归**：对应 **completeWork**

**遍历顺序：**

```
递阶段（beginWork）：
1. Card 节点
2. h3 节点
3. "你好" 文本节点

归阶段（completeWork）：
4. "你好" 文本节点
5. h3 节点
6. p 节点（兄弟节点）
7. "Big-React" 文本节点
8. "Big-React" 文本节点
9. p 节点
10. Card 节点
```

**DFS 与 BFS 的区别：**

| 遍历方式 | 说明 | 特点 |
|---------|------|------|
| **DFS（深度优先）** | 先遍历子节点，再遍历兄弟节点 | React 使用，符合组件嵌套的自然顺序 |
| **BFS（广度优先）** | 先遍历同层节点，再遍历下一层 | 适合层级遍历场景 |

**为什么 React 使用 DFS？**
- 组件的依赖关系通常是父子关系，DFS 能够先处理依赖
- 便于实现递归和归的两阶段处理
- 符合调用栈的自然顺序
