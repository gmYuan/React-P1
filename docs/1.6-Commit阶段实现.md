# React Commit 阶段实现完整指南

## 第一部分：顶层鸟瞰

### 背景问题

在 React 的渲染流程中，经过 schedule 阶段和 render 阶段后，需要执行 commit 阶段来真正地将变更应用到 DOM 上。

**核心挑战：**
- 如何切换 fiber 树（current 和 finishedWork）
- 如何执行 Placement 对应的 DOM 操作
- 如何判断是否需要执行子阶段的操作

### 宏观图

**React 内部的 3 个阶段：**

```
schedule 阶段
    ↓
render 阶段（beginWork、completeWork）
    ↓
commit 阶段（commitWork）
    ↓
    ├─ beforeMutation 阶段
    ├─ mutation 阶段（Placement）
    └─ layout 阶段
```

**commit 阶段的核心流程：**

```
commitRoot(root)
    ↓
获取 finishedWork
    ↓
判断是否需要执行操作
    ↓
    ├─ subtreeHasEffect = (finishedWork.subtreeFlags & MutationMask) !== NoFlags
    ├─ rootHasEffect = (finishedWork.flags & MutationMask) !== NoFlags
    ↓
如果存在需要执行的操作
    ↓
    ├─ beforeMutation 阶段
    ├─ mutation 阶段（Placement）
    ├─ fiber 树切换：root.current = finishedWork
    └─ layout 阶段
```

### 关键技术栈

| 技术 | 说明 |
|------|------|
| **MutationMask** | 用于判断是否需要执行 mutation 阶段的标记掩码 |
| **subtreeFlags** | 子树中所有 flags 的聚合 |
| **flags** | 当前节点的副作用标记 |
| **fiber 树切换** | 将 finishedWork 树切换为 current 树 |

## 第三部分：主要知识点

### React 内部的 3 个阶段

**1. schedule 阶段**
- 调度更新的优先级
- 决定何时开始渲染

**2. render 阶段（beginWork、completeWork）**
- beginWork：自上而下创建 Fiber 节点
- completeWork：自下而上完成工作
- 构建 workInProgress 树

**3. commit 阶段（commitWork）**
- 将 render 阶段的结果应用到 DOM 上
- 不可中断

### commit 阶段的 3 个子阶段

**1. beforeMutation 阶段**
- 在 DOM 变更之前执行
- 读取组件变更前的状态

**2. mutation 阶段**
- 执行 DOM 操作
- Placement：插入新节点
- Update：更新节点属性
- ChildDeletion：删除子节点

**3. layout 阶段**
- DOM 变更完成后执行
- 可以读取 DOM 布局
- 执行副作用（如 useLayoutEffect）

### 当前 commit 阶段要执行的任务

**1. fiber 树的切换**
```javascript
root.current = finishedWork;
```
将 workInProgress 树切换为 current 树，使其成为当前显示的版本。

**2. 执行 Placement 对应操作**

需要注意的问题：考虑如下 JSX，如果 span 含有 flag，该如何找到它？

```jsx
<App>
  <div>
    <span>只因</span>
  </div>
</App>
```

**问题：** 如何高效地找到所有需要执行操作的节点？

**解决方案：** 通过 `subtreeFlags` 和 `flags` 结合 `MutationMask` 来判断：

```typescript
const subtreeHasEffect = (finishedWork.subtreeFlags & MutationMask) !== NoFlags;
const rootHasEffect = (finishedWork.flags & MutationMask) !== NoFlags;
```

- `subtreeFlags`：子树中所有节点的 flags 聚合
- `MutationMask`：mutation 阶段关心的 flags（Placement | Update | ChildDeletion）
- 通过位运算快速判断是否存在需要执行的操作

**优势：**
- 如果 `subtreeFlags & MutationMask === NoFlags`，可以直接跳过整个子树
- 避免深度遍历所有节点，提升性能
