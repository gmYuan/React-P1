# React beginWork 实现完整指南

## 第一部分：顶层鸟瞰

### 背景问题

在 React 的 Fiber 架构中，渲染流程分为两个主要阶段：

- **Render 阶段（可中断）**：遍历 Fiber 树，为每个节点打标记（flags）
- **Commit 阶段（不可中断）**：根据标记执行 DOM 操作

**核心挑战：**
- beginWork 需要处理**不同类型**的 FiberNode（HostRoot、HostComponent、HostText 等）
- 需要正确标记**副作用 flags**（Placement、Update、ChildDeletion）
- 需要为开发环境添加**调试信息**（__DEV__ 标识）

### 宏观图

**beginWork 的工作流程：**

```
workLoop()
    ↓
performUnitOfWork(fiber)
    ↓
beginWork(fiber)  ← 当前关注的部分
    ↓ 根据 fiber.tag 分支
    ├─ HostRoot → 计算状态最新值 → 创造子fiberNode
    ├─ HostComponent → 创造子fiberNode
    └─ HostText → 无 beginWork 工作流程（因为没有子节点）
    ↓
标记 flags
    ↓
返回 child（继续遍历）或 null（进入 completeWork）
```

**关键技术栈：**

| 技术 | 说明 |
|------|------|
| **__DEV__** | 开发环境标识，用于打印调试信息 |
| **flags** | 副作用标记（Placement、ChildDeletion、Update） |
| **@rollup/plugin-replace** | Rollup 插件，用于替换代码中的变量 |

### 步骤 2：创建并实现 beginWork.ts

**路径：** `packages/react-reconciler/src/beginWork.ts`（新建文件）

**说明：** 实现 Render 阶段的 beginWork，根据不同的 fiber 类型处理子节点的创建和协调

**完整代码：**

```typescript
import { FiberNode } from './fiber';
import { HostComponent, HostRoot, HostText } from './workTags';
import { processUpdateQueue, UpdateQueue } from './updateQueue';
import { ReactElementType } from 'shared/ReactTypes';
import { mountChildFibers, reconcileChildFibers } from './childFibers';

// 递归中的递阶段
export const beginWork = (wip: FiberNode) => {
  // 比较,返回子fiberNode
  switch (wip.tag) {
    case HostRoot:
      return updateHostRoot(wip);
    case HostComponent:
      return updateHostComponent(wip);
    case HostText:
      return null;
    default:
      if (__DEV__) {
        console.warn('beginWork未实现的类型');
      }
      break;
  }
};

function updateHostRoot(wip: FiberNode) {
  const baseState = wip.memoizedState;
  const updateQueue = wip.updateQueue as UpdateQueue<Element>;
  const pending = updateQueue.shared.pending;
  updateQueue.shared.pending = null;
  const { memoizedState } = processUpdateQueue(baseState, pending);
  wip.memoizedState = memoizedState;

  const nextChildren = wip.memoizedState;
  reconcileChildren(wip, nextChildren);
  return wip.child;
}

function updateHostComponent(wip: FiberNode) {
  const nextProps = wip.pendingProps;
  const nextChildren = nextProps.children;
  reconcileChildren(wip, nextChildren);
  return wip.child;
}

function reconcileChildren(wip: FiberNode, children?: ReactElementType) {
  const current = wip.alternate;
  if (current !== null) {
    // update
    wip.child = reconcileChildFibers(wip, current.child, children);
  } else {
    // mount
    wip.child = mountChildFibers(wip, null, children);
  }
}
```

### 步骤 3：创建并实现 childFibers.ts

**路径：** `packages/react-reconciler/src/childFibers.ts`（新建文件）

**说明：** 实现子节点协调逻辑，根据新旧节点对比生成新的 fiber 节点并标记副作用

**完整代码：**

```typescript
import { ReactElementType } from 'shared/ReactTypes';
import { createFiberFromElement, FiberNode } from './fiber';
import { REACT_ELEMENT_TYPE } from 'shared/ReactSymbols';
import { HostText } from './workTags';
import { Placement } from './fiberFlags';

function ChildReconciler(shouldTrackEffects: boolean) {
  function reconcileSingleElement(
    returnFiber: FiberNode,
    currentFiber: FiberNode | null,
    element: ReactElementType
  ) {
    // 根据 element 创建fiber
    const fiber = createFiberFromElement(element);
    fiber.return = returnFiber;
    return fiber;
  }

  function reconcileSingleTextNode(
    returnFiber: FiberNode,
    currentFiber: FiberNode | null,
    content: string | number
  ) {
    const fiber = new FiberNode(HostText, { content }, null);
    fiber.return = returnFiber;
    return fiber;
  }

  function placeSingleChild(fiber: FiberNode) {
    if (shouldTrackEffects && fiber.alternate === null) {
      fiber.flags |= Placement;
    }
    return fiber;
  }

  return function reconcileChildFibers(
    returnFiber: FiberNode,
    currentFiber: FiberNode | null,
    newChild?: ReactElementType | undefined
  ) {
    // 判断当前fiber的类型
    if (typeof newChild === 'object' && newChild !== null) {
      switch (newChild.$$typeof) {
        case REACT_ELEMENT_TYPE:
          return placeSingleChild(
            reconcileSingleElement(returnFiber, currentFiber, newChild)
          );
        default:
          if (__DEV__) {
            console.warn('未实现的reconcile类型', newChild);
          }
          break;
      }
    }

    // TODO 多节点的情况 ul> li*3

    // HostText
    if (typeof newChild === 'string' || typeof newChild === 'number') {
      return placeSingleChild(
        reconcileSingleTextNode(returnFiber, currentFiber, newChild)
      );
    }

    if (__DEV__) {
      console.warn('未实现的reconcile类型', newChild);
    }

    return null;
  };
}

export const reconcileChildFibers = ChildReconciler(true);
export const mountChildFibers = ChildReconciler(false);
```

## 第三部分：主要知识点

### 知识点 1：beginWork

**对于如下结构的 reactElement：**

```jsx
<A>
<B/>
</A>
```

**描述：** 当进入 A 的 beginWork 时，通过对比 B `current` fiberNode 与 B `reactElement`，生成 B 对应 wip fiberNode。

在此过程中最多会标记 2 类与「结构变化」相关的 flags：

- **Placement**

  插入：`a -> ab` 移动：`abc -> bca`

- **ChildDeletion**

  删除：`ul>li*3 -> ul>li*1`

不包含与「属性变化」相关的 flag：

- **Update**

  `<img title="鸡" /> -> <img title="你太美" />`

---

### 知识点 2：实现与 Host 相关节点的 beginWork

**首先，为开发环境增加 __DEV__ 标识，方便 Dev 包打印更多信息：**

```javascript
// @rollup/plugin-replace
```

**HostRoot 的 beginWork 工作流程：**

1. 计算状态的最新值
2. 创造子 fiberNode

**HostComponent 的 beginWork 工作流程：**

1. 创造子 fiberNode

**HostText 没有 beginWork 工作流程（因为他没有子节点）**

---

### 知识点 3：beginWork 性能优化策略

**考虑如下结构的 reactElement：**

```jsx
<div>
  <p>练习时长</p>
  <span>两年半</span>
</div>
```

**理论上 mount 流程完毕后包含的 flags：**

- 两年半 Placement
- span Placement
- 练习时长 Placement
- p Placement
- div Placement

相比于执行 5 次 Placement，我们可以构建好「离屏 DOM 树」后，对 div 执行 1 次 Placement 操作。

## 第四部分：个人 Q&A

### Q1: 刚刚的改动主要做了哪些事情，实现了哪些功能？项目的入口是什么？

**一句话概括：**
我们实现了 `beginWork` 阶段 —— React Fiber 树遍历的"向下递"过程，让 React 知道如何处理不同类型的节点并创建它们的子 Fiber。

**类比理解：**
就像工厂流水线的"拆箱工序" —— 拆开一个大包裹（父节点），检查里面是什么类型的物品（HostRoot/HostComponent/HostText），然后为每个物品准备专门的处理流程，并创建子包裹（子 Fiber）。

#### 完整的调用链

```
【之前已实现 ✅】
createContainer → updateContainer → scheduleUpdateOnFiber → renderRoot → workLoop
    ↓
【本次实现 ✅】
performUnitOfWork(wip)
    ↓
beginWork(wip)  ← 🔴 新增：处理 Fiber 节点
    ↓
    ├─ HostRoot → updateHostRoot → reconcileChildren
    ├─ HostComponent → updateHostComponent → reconcileChildren  
    └─ HostText → null（文本节点没有子节点）
    ↓
reconcileChildren(wip, children)  ← 🔴 新增：创建子 Fiber
    ↓
childFibers.reconcileChildFibers(...)  ← 🔴 新增：对比并标记副作用
    ↓
    ├─ 单个 ReactElement → reconcileSingleElement → createFiberFromElement
    ├─ 文本内容 → reconcileSingleTextNode → new FiberNode(HostText)
    └─ placeSingleChild → 标记 Placement 标记
    ↓
返回 child（子 Fiber）给 performUnitOfWork
    ↓
【待实现】
completeUnitOfWork → completeWork（处理"归"阶段）
```

#### 本次改动的核心文件

1. **`beginWork.ts`**（核心）：根据 Fiber 类型分发处理
2. **`childFibers.ts`**（核心）：创建子 Fiber 并标记副作用
3. **`fiber.ts`**：新增 `createFiberFromElement` 
4. **`utils.js`**：配置 `__DEV__` 标识
5. **`reconciler.d.ts`**：声明 `__DEV__` 类型

#### 总结

我们实现了 Fiber 树遍历的"递"阶段 —— `beginWork` 负责向下处理每个节点，为不同类型的节点（根节点、普通元素、文本）创建子 Fiber，并通过 `childFibers` 标记哪些节点需要插入到 DOM（Placement）。

当前还没有实现"归"阶段（`completeWork`），那是处理 DOM 属性和收集副作用的工作。
