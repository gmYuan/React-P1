# React useState 的 mount 时实现完整指南

## 第一部分：顶层鸟瞰

### 背景问题

在 React Hooks 中，useState 是最常用的 Hook 之一。在组件首次渲染（mount）时，需要：
- 创建 Hook 数据结构
- 初始化 state 值
- 创建 dispatch 函数并绑定到当前 fiber 和 updateQueue
- 建立 Hook 链表结构

**核心挑战：**
- 如何存储和管理 Hook 数据（使用 Hook 对象和链表）
- 如何创建 dispatch 函数并正确绑定上下文
- 如何将 Hook 挂载到 FiberNode 上
- 如何处理组件中的多个 Hook（通过链表串联）

### 宏观图

**useState 在 mount 时的核心流程：**

```
用户调用 useState(initialState)
         ↓
   mountState 函数
         ↓
 mountWorkInProgressHook (创建 Hook 对象)
         ↓
   ┌────┴────┐
   ↓         ↓
第一个Hook  后续Hook
   ↓         ↓
挂到fiber   加到链表
.memoizedState  .next
         ↓
   初始化 memoizedState
         ↓
   创建 updateQueue
         ↓
   创建 dispatch 函数
   (dispatchSetState.bind)
         ↓
   dispatch 挂载到 queue
         ↓
   返回 [state, dispatch]
```

**数据结构关系：**

```
FiberNode
  └─ memoizedState → Hook (第一个)
                       ├─ memoizedState: state 值
                       ├─ updateQueue: 更新队列
                       │    └─ dispatch: setState 函数
                       └─ next → Hook (第二个)
                              ├─ memoizedState
                              ├─ updateQueue
                              └─ next → ...
```

### 关键技术栈

| 技术 | 说明 |
|------|------|
| **Hook 对象** | 存储单个 Hook 的数据结构（memoizedState、updateQueue、next） |
| **Hook 链表** | 通过 next 指针串联多个 Hook，挂载在 fiber.memoizedState 上 |
| **workInProgressHook** | 指向当前正在处理的 Hook，用于构建链表 |
| **currentlyRenderingFiber** | 当前正在渲染的 FiberNode，用于绑定 dispatch |
| **dispatchSetState** | setState 的实际实现，通过 bind 绑定 fiber 和 queue |
| **UpdateQueue** | 更新队列，包含 dispatch 字段用于内部共享 |

```diff
+ const HooksDispatcherOnMount: Dispatcher = {
+   useState: mountState
+ };

+ function mountState<State>(
+   initialState: (() => State) | State
+ ): [State, Dispatch<State>] {
+   // 找到当前useState对应的hook数据
+   const hook = mountWorkInProgressHook();
+   let memoizedState;
+   if (initialState instanceof Function) {
+     memoizedState = initialState();
+   } else {
+     memoizedState = initialState;
+   }
+   
+   const queue = createUpdateQueue<State>();
+   hook.updateQueue = queue;
+   hook.memoizedState = memoizedState;
+   
+   // @ts-ignore
+   const dispatch = dispatchSetState.bind(null, currentlyRenderingFiber, queue);
+   queue.dispatch = dispatch;
+   
+   return [memoizedState, dispatch];
+ }

+ function mountWorkInProgressHook(): Hook {
+   const hook: Hook = {
+     memoizedState: null,
+     updateQueue: null,
+     next: null
+   };
+   
+   if (workInProgressHook === null) {
+     // mount时 第一个hook
+     if (currentlyRenderingFiber === null) {
+       throw new Error('请在函数组件内调用hook');
+     } else {
+       workInProgressHook = hook;
+       currentlyRenderingFiber.memoizedState = workInProgressHook;
+     }
+   } else {
+     // mount时 后续的hook
+     workInProgressHook.next = hook;
+     workInProgressHook = hook;
+   }
+   
+   return workInProgressHook;
+ }

+ function dispatchSetState<State>(
+   fiber: FiberNode,
+   updateQueue: UpdateQueue<State>,
+   action: Action<State>
+ ) {
+   const update = createUpdate(action);
+   enqueueUpdate(updateQueue, update);
+   scheduleUpdateOnFiber(fiber);
+ }
```

### Q1：理解 useState 的基本需求

#### 问题1：useState 需要返回什么？

用户这样使用：
```javascript
function App() {
  const [count, setCount] = useState(0);
  return <div onClick={() => setCount(count + 1)}>{count}</div>;
}
```

<details>
<summary>答案</summary>

需要返回两个东西：
1. **当前的值**（count = 0）
2. **修改值的函数**（setCount）

</details>

---

#### 问题2：当用户调用 `setCount(1)` 后，下次渲染时 `count` 应该是多少？

<details>
<summary>答案</summary>

应该是 1，也就是说 **useState 需要记住这个值**

</details>

---

#### 问题3：如果一个组件里有多个 useState，React 怎么知道哪个值对应哪个 useState？

```javascript
function App() {
  const [name, setName] = useState('张三');     // 第1个
  const [age, setAge] = useState(18);           // 第2个
  const [city, setCity] = useState('北京');     // 第3个
}
```

<details>
<summary>答案</summary>

**React 使用调用顺序来区分！**

- 第1次调用 useState → 对应第1个值（name）
- 第2次调用 useState → 对应第2个值（age）
- 第3次调用 useState → 对应第3个值（city）

**所以 React 需要：**
1. 用一个**链表**把这些值串起来
2. 记住每次调用的顺序

这就是为什么 Hook 不能在 if 语句里使用！如果顺序变了，就对不上了。

</details>

---

### Q2：React 用什么数据结构来实现？

#### Hook 对象 - 存储单个 useState 的数据

```typescript
interface Hook {
  memoizedState: any;      // 存储当前值（比如 count = 0）
  updateQueue: unknown;    // 存储更新队列（暂时不用关心）
  next: Hook | null;       // 指向下一个 Hook（用来串联多个 useState）
}
```

**示例：** 假设有 3 个 useState：

```javascript
const [name, setName] = useState('张三');
const [age, setAge] = useState(18);
const [city, setCity] = useState('北京');
```

每个 useState 会创建一个 Hook 对象：

```
Hook1: { memoizedState: '张三', next: → Hook2 }
Hook2: { memoizedState: 18,     next: → Hook3 }
Hook3: { memoizedState: '北京', next: null }
```

#### Hook 存储在哪里？

每个组件（FiberNode）有一个 `memoizedState` 字段，指向第一个 Hook：

```
FiberNode (App 组件)
  └─ memoizedState → Hook1 → Hook2 → Hook3 → null
```

**总结：**
1. 一个 Hook 对象存储了：**当前值** 和 **指向下一个 Hook 的指针**
2. 多个 Hook 根据调用时书写的顺序，通过 `next` 链表进行关联

### Q3：mountState 函数的工作流程

当用户第一次调用 `useState(0)` 时，会执行 `mountState` 函数。

#### 步骤1：创建 Hook 对象

```typescript
const hook = mountWorkInProgressHook();
```

**这行代码做了什么？**

<details>
<summary>答案</summary>

创建一个新的 Hook 对象，并把它加入到链表中：
- 如果是第1个 Hook → 挂到 `fiber.memoizedState`
- 如果是第2、3个... → 挂到上一个 Hook 的 `next`

</details>

---

#### 步骤2：初始化值

```typescript
let memoizedState;
if (initialState instanceof Function) {
  memoizedState = initialState();
} else {
  memoizedState = initialState;
}
```

**为什么要检查 `initialState instanceof Function`？**

<details>
<summary>答案</summary>

因为 useState 支持两种用法：

```javascript
// 方式1：直接传值
const [count, setCount] = useState(0);

// 方式2：传函数
const [count, setCount] = useState(() => {
  const savedValue = localStorage.getItem('count');
  return savedValue ? Number(savedValue) : 0;
});
```

**区别：**
- **方式1（直接传值）：** 每次渲染都会执行 `0` 这个值（虽然很快）
- **方式2（传函数）：** 函数只在**第一次渲染**时执行一次

**好处：** 如果初始值的计算很复杂（比如从 localStorage 读取、复杂计算），用函数形式可以避免重复计算，提升性能。

</details>

---

#### 步骤3：保存值到 Hook 对象

```typescript
const queue = createUpdateQueue<State>();
hook.updateQueue = queue;
hook.memoizedState = memoizedState;  // 把初始值保存起来
```

这里做了两件事：
1. 创建一个更新队列（用于后续的 `setCount` 更新）
2. **把初始值保存到 `hook.memoizedState`**

return [memoizedState, dispatch];

### 回到代码

**原始的 dispatchSetState 函数需要3个参数：**

```typescript
function dispatchSetState(fiber, updateQueue, action) {
  // fiber: 哪个组件要更新
  // updateQueue: 哪个 useState 要更新
  // action: 新的值是什么
}
```

**用户调用时只想传1个参数：**

```javascript
setCount(1);  // 用户只想告诉 React 新值是 1
```

**bind 的作用就是"预先填入"前面的参数：**

```typescript
const dispatch = dispatchSetState.bind(null, currentlyRenderingFiber, queue);
//                                            ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//                                            预先填入"哪个组件"和"哪个useState"

// 现在 dispatch 变成了：
dispatch = function(action) {
  dispatchSetState(已知的fiber, 已知的queue, action);
}
```

---

### 为什么需要这样做？

用户调用 `setCount(1)` 时，React 需要知道：
1. 是**哪个组件**要更新？（可能有100个组件都用了 useState）
2. 是组件里的**哪个 useState**？（一个组件可能有多个 useState）

通过 `bind` 预先绑定：
- **fiber**：就是当前组件（currentlyRenderingFiber）
- **queue**：就是这个特定 Hook 的更新队列

这样用户调用 `setCount(1)` 时，React 就知道：
- 要更新**这个组件**
- 要更新这个组件的**这个 useState**
- 新值是 **1**

</details>

### 在 dispatchSetState 中为什么传 null？

看看 `dispatchSetState` 函数：

```typescript
function dispatchSetState<State>(
  fiber: FiberNode,
  updateQueue: UpdateQueue<State>,
  action: Action<State>
) {
  const update = createUpdate(action);
  enqueueUpdate(updateQueue, update);
  scheduleUpdateOnFiber(fiber);
  // ↑ 注意：这里没有用到 this
}
```

**关键点：** 这个函数内部**没有使用 `this`**！

所以：
- `bind` 的第一个参数（this）传什么都无所谓
- 传 `null` 表示"我不需要 this"

---

### 完整理解

```typescript
const dispatch = dispatchSetState.bind(null, currentlyRenderingFiber, queue);
//                                     ^^^^  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//                                     this   预填充的参数
//                                   (不需要)  (重要！)
```

**等价于：**

```typescript
const dispatch = function(action) {
  // this 是 null，但我们不用 this，所以无所谓
  dispatchSetState(currentlyRenderingFiber, queue, action);
};
```

---

**总结：**
- `bind(null, ...)` 中的 `null` 是给 `this` 用的
- 因为 `dispatchSetState` 函数里不需要用 `this`
- 所以第一个参数传 `null` 就够了
- **bind 的真正作用是预先填入后面的参数（fiber 和 queue）**

</details>

### Q3.5：dispatchSetState 函数详解

当用户调用 `setCount(1)` 时，实际执行的是 `dispatchSetState` 函数。

#### 函数代码

```typescript
function dispatchSetState<State>(
  fiber: FiberNode,
  updateQueue: UpdateQueue<State>,
  action: Action<State>
) {
  const update = createUpdate(action);
  enqueueUpdate(updateQueue, update);
  scheduleUpdateOnFiber(fiber);
}
```

---

#### 第1步：创建 update 对象

```typescript
const update = createUpdate(action);
```

**这行做了什么？**

<details>
<summary>答案</summary>

创建一个 update 对象，包含用户传入的新值（action）。

**createUpdate 函数（在 updateQueue.ts 中）：**

```typescript
export const createUpdate = <State>(action: Action<State>): Update<State> => {
  return {
    action  // 用户传入的值，比如 1 或 (prev) => prev + 1
  };
};
```

**例子：**
```javascript
setCount(1);           // action = 1
setCount(x => x + 1);  // action = (x) => x + 1
```

</details>

---

#### 第2步：将 update 加入队列

```typescript
enqueueUpdate(updateQueue, update);
```

**这行做了什么？**

<details>
<summary>答案</summary>

把 update 对象放入 updateQueue 的 pending 队列中。

**enqueueUpdate 函数（在 updateQueue.ts 中）：**

```typescript
export const enqueueUpdate = <State>(
  updateQueue: UpdateQueue<State>,
  update: Update<State>
) => {
  updateQueue.shared.pending = update;
};
```

**为什么要放入队列？**

因为可能在同一次渲染中多次调用 setState：

```javascript
function handleClick() {
  setCount(1);
  setCount(2);
  setCount(3);
}
```

这些更新需要排队处理，所以放入 `updateQueue.shared.pending` 中。

**注意：** 目前的实现是简化版，直接覆盖。实际 React 中是环形链表，可以存储多个 update。

</details>

---

#### 第3步：触发更新调度

```typescript
scheduleUpdateOnFiber(fiber);
```

**这行做了什么？**

<details>
<summary>答案</summary>

通知 React："这个组件需要重新渲染了！"

这个函数会：
1. 标记这个 fiber 需要更新
2. 触发 React 的调度机制
3. 在合适的时机重新渲染这个组件

**简化理解：**
```
scheduleUpdateOnFiber(fiber)
  ↓
React 调度器
  ↓
重新调用 renderWithHooks(fiber)
  ↓
重新执行组件函数
  ↓
useState 会读取 updateQueue 中的 update
  ↓
计算新的 state 值
  ↓
组件重新渲染
```

</details>

#### 完整流程示例

假设用户点击按钮，从 0 变成 1：

```javascript
function App() {
  const [count, setCount] = useState(0);
  
  return (
    <div onClick={() => setCount(1)}>
      {count}
    </div>
  );
}
```

**执行流程：**

<details>
<summary>点击查看详细流程</summary>

**1. 用户点击，调用 setCount(1)**

```
setCount(1)
  ↓ (实际调用的是 bind 后的 dispatchSetState)
dispatchSetState(App组件的fiber, Hook的queue, 1)
```

**2. dispatchSetState 执行**

```typescript
// 步骤1: 创建 update
const update = createUpdate(1);
// update = { action: 1 }

// 步骤2: 加入队列
enqueueUpdate(queue, update);
// queue.shared.pending = { action: 1 }

// 步骤3: 触发调度
scheduleUpdateOnFiber(App组件的fiber);
```

**3. React 重新渲染组件**

```
renderWithHooks(App组件)
  ↓
执行 App()
  ↓
调用 useState(0)
  ↓
这次不是 mount，而是 update
  ↓
从 updateQueue 中取出 update
  ↓
计算新值: 
  - 如果 action 是值: newState = 1
  - 如果 action 是函数: newState = action(oldState)
  ↓
返回 [1, setCount]
  ↓
组件用新值渲染
```

</details>

#### 补充：queue 是谁的？

**重要：queue 是 Hook 对象上的，不是 Fiber 上的！**

<details>
<summary>点击查看详细说明</summary>

**数据结构层级：**

```
FiberNode (组件)
  └─ memoizedState → Hook (第一个 useState)
                       │
                       ├─ memoizedState: 0
                       ├─ updateQueue: queue ← 就是这个 queue！
                       │    └─ shared: { pending: null }
                       │    └─ dispatch: setCount
                       └─ next: Hook2
```

**每个 Hook 都有自己的 queue：**

```
Hook1 (第1个 useState)
  ├─ memoizedState: '张三'
  └─ updateQueue: queue1  ← 第1个 useState 的更新队列

Hook2 (第2个 useState)
  ├─ memoizedState: 18
  └─ updateQueue: queue2  ← 第2个 useState 的更新队列

Hook3 (第3个 useState)
  ├─ memoizedState: '北京'
  └─ updateQueue: queue3  ← 第3个 useState 的更新队列
```

</details>

#### queue 的双重作用

**问题1：** 一个组件有多个 useState，怎么知道是哪个 setState 被调用了？

<details>
<summary>答案</summary>

**通过 bind 绑定不同的 queue 来区分！**

```javascript
function App() {
  const [name, setName] = useState('张三');
  const [age, setAge] = useState(18);
  
  setName('李四');  // ← 绑定的是 Hook1 的 queue1
  setAge(20);       // ← 绑定的是 Hook2 的 queue2
}
```

```typescript
// setName 绑定 Hook1 的 queue1
const setName = dispatchSetState.bind(null, fiber, queue1);

// setAge 绑定 Hook2 的 queue2  
const setAge = dispatchSetState.bind(null, fiber, queue2);
```

</details>

---

**问题2：** 某个 setState 被多次调用了，怎么处理？

<details>
<summary>答案</summary>

**通过 queue 存储多个 update！**

```javascript
function handleClick() {
  setCount(1);
  setCount(2);
  setCount(3);
}
```

理论上 queue 内部应该是链表，存储多个 update：

```
Hook1
  └─ updateQueue: queue1
       └─ shared.pending: update1 → update2 → update3
```

**注意：** 目前的简化实现是直接覆盖，完整的 React 实现会用环形链表串联多个 update。

</details>

---

#### 关键要点总结

**queue 的双重作用：**

| 作用 | 通过什么机制 | 解决什么问题 |
|------|------------|------------|
| **区分不同的 useState** | 每个 Hook 有自己的 queue，通过 bind 绑定 | 知道是哪个 setState 被调用了 |
| **存储多次更新** | queue 内部存储多个 update（应该是链表） | 处理同一个 setState 被多次调用 |

**三步走：**
1. 创建 update 对象
2. 加入更新队列
3. 触发组件重新渲染

**action 的两种形式：**
```javascript
setCount(1);           // 直接值
setCount(x => x + 1);  // 函数（可以基于旧值计算）
```

### Q4：理解 mountWorkInProgressHook（创建 Hook 链表）

这个函数负责创建 Hook 对象并加入链表。

#### 第1段：创建 Hook 对象

```typescript
const hook: Hook = {
  memoizedState: null,
  updateQueue: null,
  next: null
};
```

这很简单，就是创建一个空的 Hook 对象。

---

#### 第2段：判断是第几个 Hook

```typescript
if (workInProgressHook === null) {
  // 第一个 hook
} else {
  // 后续的 hook
}
```

**问题：`workInProgressHook` 是什么？它是用来干什么的？**

<details>
<summary>答案</summary>

`workInProgressHook` 是一个**全局变量**，指向**当前正在处理的 Hook**。

```typescript
let workInProgressHook: Hook | null = null;
```

- 初始值是 `null`
- 每创建一个 Hook，就更新为指向最新的那个 Hook
- 用来维护 Hook 链表

</details>

---

#### 第3段：第一个 Hook 的处理

```typescript
if (workInProgressHook === null) {
  // mount时 第一个hook
  if (currentlyRenderingFiber === null) {
    throw new Error('请在函数组件内调用hook');
  } else {
    workInProgressHook = hook;
    currentlyRenderingFiber.memoizedState = workInProgressHook;
  }
}
```

**这段代码做了什么？**

当这是**第一个 Hook** 时：
1. 先检查 `currentlyRenderingFiber` 是否为空（确保在组件内调用）
2. 把新创建的 `hook` 赋值给 `workInProgressHook`
3. **把这个 Hook 挂到组件的 `memoizedState` 上**

**图解：**

```
组件首次渲染，调用第一个 useState：

FiberNode (组件)
  └─ memoizedState: null  →  指向 Hook1

workInProgressHook: null  →  指向 Hook1
```

---

#### 为什么要检查 currentlyRenderingFiber === null？

<details>
<summary>点击查看详细解释</summary>

**currentlyRenderingFiber 是什么？**

```typescript
let currentlyRenderingFiber: FiberNode | null = null;
```

这是一个**全局变量**，表示**当前正在渲染的组件**。

---

**它什么时候有值？什么时候是 null？**

看 `renderWithHooks` 函数：

```typescript
export function renderWithHooks(wip: FiberNode) {
  // 渲染开始：设置当前组件
  currentlyRenderingFiber = wip;  // ← 有值了！
  
  const Component = wip.type;
  const props = wip.pendingProps;
  const children = Component(props);  // ← 执行函数组件，这时调用 useState
  
  // 渲染结束：重置为 null
  currentlyRenderingFiber = null;  // ← 又变成 null 了
  return children;
}
```

**流程：**

```
1. 渲染开始
   currentlyRenderingFiber = 当前组件 ✓

2. 执行组件函数
   function App() {
     const [count, setCount] = useState(0);  // ← 这时 currentlyRenderingFiber 有值
   }

3. 渲染结束
   currentlyRenderingFiber = null ✗
```

---

**为什么为 null 时要报错？**

**问题场景：** 如果用户在组件外面调用 Hook：

```javascript
// ❌ 错误用法：在组件外调用
const [count, setCount] = useState(0);

function App() {
  return <div>{count}</div>;
}
```

这时：
- `renderWithHooks` 还没执行
- `currentlyRenderingFiber` 还是 `null`
- Hook 不知道应该挂到哪个组件上！

---

**总结：**

这个检查是为了**防止用户在错误的地方调用 Hook**：

- ✅ 只能在函数组件内部调用
- ❌ 不能在组件外部调用
- ❌ 不能在普通函数里调用
- ❌ 不能在 class 组件里调用

**如果 `currentlyRenderingFiber` 是 null，说明现在不在渲染任何组件，Hook 无法知道应该挂载到哪里，所以必须报错！**

</details>

---

#### 第4段：后续 Hook 的处理

```typescript
else {
  // mount时 后续的hook
  workInProgressHook.next = hook;
  workInProgressHook = hook;
}
```

**这段代码做了什么？**

当这是**第2、3、4...个 Hook** 时：
1. 把新 Hook 挂到上一个 Hook 的 `next` 上
2. 更新 `workInProgressHook` 指向最新的 Hook

**图解：**

```
调用第二个 useState：

之前：
Hook1 (workInProgressHook 指向这里)
  └─ next: null

之后：
Hook1 
  └─ next: → Hook2 (workInProgressHook 现在指向这里)
```

---

#### 完整流程演示

假设组件有 3 个 useState：

```javascript
function App() {
  const [name, setName] = useState('张三');     // 调用1
  const [age, setAge] = useState(18);           // 调用2
  const [city, setCity] = useState('北京');     // 调用3
}
```

**执行过程：**

<details>
<summary>点击查看详细过程</summary>

**调用1：** `useState('张三')`
```
1. workInProgressHook = null (第一个Hook)
2. 创建 Hook1
3. fiber.memoizedState = Hook1
4. workInProgressHook = Hook1

结果：
FiberNode
  └─ memoizedState → Hook1
                      └─ next: null
```

**调用2：** `useState(18)`
```
1. workInProgressHook = Hook1 (不是null，是后续Hook)
2. 创建 Hook2
3. Hook1.next = Hook2
4. workInProgressHook = Hook2

结果：
FiberNode
  └─ memoizedState → Hook1 → Hook2
                              └─ next: null
```

**调用3：** `useState('北京')`
```
1. workInProgressHook = Hook2 (不是null，是后续Hook)
2. 创建 Hook3
3. Hook2.next = Hook3
4. workInProgressHook = Hook3

结果：
FiberNode
  └─ memoizedState → Hook1 → Hook2 → Hook3
                                      └─ next: null
```

</details>

---

**总结：**

- 第一个 Hook：挂到 `fiber.memoizedState`
- 后续 Hook：挂到上一个 Hook 的 `next`
- `workInProgressHook` 始终指向最新的那个 Hook

### Q5：useState 完整流程总结

#### 用户代码示例

```javascript
function App() {
  const [count, setCount] = useState(0);
  return <div onClick={() => setCount(1)}>{count}</div>;
}
```

#### 内部执行流程

<details>
<summary>点击查看完整流程</summary>

**1. renderWithHooks 开始渲染组件**
```typescript
// 设置全局变量
currentlyRenderingFiber = App组件
currentDispatcher.current = HooksDispatcherOnMount
```

**2. 调用 useState(0)**
```typescript
// 实际调用
mountState(0)
```

**3. mountState 执行**
```typescript
// 步骤1: 创建 Hook 并加入链表
const hook = mountWorkInProgressHook();

// 步骤2: 处理初始值（支持函数形式）
let memoizedState;
if (initialState instanceof Function) {
  memoizedState = initialState();
} else {
  memoizedState = initialState;  // = 0
}

// 步骤3: 创建 updateQueue 并保存值
const queue = createUpdateQueue<State>();
hook.updateQueue = queue;
hook.memoizedState = memoizedState;

// 步骤4: 用 bind 创建 dispatch 函数
const dispatch = dispatchSetState.bind(null, currentlyRenderingFiber, queue);
queue.dispatch = dispatch;

// 步骤5: 返回结果
return [0, setCount];
```

**4. 用户点击触发 setCount(1)**
```typescript
// 实际调用（已经通过 bind 绑定了 fiber 和 queue）
dispatchSetState(App组件, queue, 1)
  ↓
创建 update 并加入队列
  ↓
触发组件更新
```

</details>

---

#### 核心数据结构

```
FiberNode (App组件)
  └─ memoizedState → Hook1 → Hook2 → Hook3
                       │
                       ├─ memoizedState: 0          (当前值)
                       ├─ updateQueue: {             (更新队列)
                       │    shared: { pending: null },
                       │    dispatch: setCount       (setState 函数)
                       │  }
                       └─ next: Hook2                (下一个 Hook)
```

---

#### 关键要点总结

**1. Hook 链表**
- 多个 Hook 通过 `next` 指针串联成链表
- 第一个 Hook 挂在 `fiber.memoizedState` 上
- 通过调用顺序来区分不同的 Hook

**2. 全局变量的作用**
- `currentlyRenderingFiber`：当前正在渲染的组件
- `workInProgressHook`：当前正在处理的 Hook（用于构建链表）

**3. bind 的作用**
- 预先绑定 `fiber` 和 `queue` 参数
- 让用户只需传入新值（action）
- dispatch 函数知道要更新哪个组件的哪个 Hook

**4. 为什么 Hook 不能在 if 里使用？**
- Hook 通过调用顺序来区分
- 如果顺序变了，Hook 链表就对不上了
- 会导致取到错误的值
